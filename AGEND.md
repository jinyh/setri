# AGEND.md

## 1. 文档定位

本文件是本仓库的双引擎协作章程，适用于 Codex 与 Claude Code。

- Codex 是默认主执行引擎。
- Claude Code 是默认辅助引擎。
- 本文件负责定义主辅关系、任务路由、介入条件、交接格式和冲突决策。
- 项目背景、当前阶段、关键文件和业务事实不在本文件展开，优先查看 [CLAUDE.md](./CLAUDE.md) 与 [docs/prd.md](./docs/prd.md)。

## 2. 决策优先级

发生规则冲突时，按以下顺序执行：

1. 用户的直接指令
2. 本文件 `AGEND.md`
3. [CLAUDE.md](./CLAUDE.md)
4. 其他项目文档与历史说明

发生事实冲突时，按以下顺序确认：

1. 当前任务中用户明确给出的最新事实
2. [docs/prd.md](./docs/prd.md)
3. [CLAUDE.md](./CLAUDE.md)
4. 其他补充文档

未经确认，不得自行发明项目事实或扩展业务边界。

## 3. 引擎角色

### Codex

Codex 负责默认承接和主推进：

- 任务拆解与执行路径判断
- 代码实现、文档落地、脚本编写和工程改动
- 本地验证、结果汇总和最终交付
- 双引擎协作时的主线控制与结论收口

### Claude Code

Claude Code 负责辅助支持与关键节点复核：

- 复杂方案评审与替代路径分析
- 架构调整前的补充审阅
- 关键文档的逻辑检查与盲点发现
- 验收、合并或大范围修改前的复核
- Codex 连续受阻时的兜底支持

默认情况下，Claude Code 不接管主执行，不单独定义最终方案，不替代 Codex 做统一收口。

## 4. 任务路由规则

默认所有任务先由 Codex 接手。

以下任务由 Codex 单引擎直接完成：

- 小型代码修改
- 常规 bug 修复
- 普通脚本或工具编写
- 已有方案下的实现工作
- 一般性文档更新

以下任务必须引入 Claude Code 参与：

- 需求存在明显歧义，且影响方案选择
- 涉及架构级调整、模块边界重划或大规模重构
- 需要对 PRD、设计文档、协作章程做结构性修改
- 验收前、合并前或对外展示前需要独立复核
- Codex 连续两轮尝试仍未消除关键不确定性

以下任务建议引入 Claude Code：

- 跨前后端的大变更
- 高风险数据迁移或目录结构调整
- 需要多方案比较的设计类任务
- 需要补充“反方视角”审查的实现任务

## 5. 协作方式

当 Claude Code 介入时，Codex 必须提供最小完整交接包，至少包括：

- 任务目标
- 当前状态
- 已确认事实来源
- 已尝试方案
- 剩余风险和待决问题
- 期望 Claude Code 输出的具体内容

Claude Code 的返回结果默认属于以下几类之一：

- 方案评审意见
- 风险清单
- 替代实现建议
- 文档修订建议
- 验收复核结论

双引擎协作结束后，由 Codex 统一吸收结果并输出最终结论。不得把两个引擎的原始结论并列留给用户自行裁决，除非用户明确要求保留分歧。

## 6. 交付要求

Codex 作为主执行引擎，必须保证以下事项：

- 最终输出与仓库事实一致
- 最终修改经过必要验证
- 引入 Claude Code 时说明介入原因
- 吸收辅助结论后完成统一收口

Claude Code 作为辅助引擎，必须遵守以下边界：

- 不重写已确认的用户目标
- 不擅自扩大任务范围
- 不替代 Codex 进行主线实施决策
- 以补充、复核、挑战盲点为主，不重复主执行工作

各文件职责固定如下：

- [AGEND.md](./AGEND.md)

## 8. 默认执行口径

本仓库后续协作默认采用以下口径：

- Codex 为主，Claude Code 为辅
- 优先单引擎推进，命中触发条件再升级为双引擎协作
- 项目事实先看 PRD 和项目上下文，再做方案判断
- 文档、代码、验证和最终交付由 Codex 统一收口

如无用户明确改写，本文件长期有效。
