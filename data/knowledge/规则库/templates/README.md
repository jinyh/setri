# 规则模板使用指南

本目录包含两种规则模板，用于编写规则库中的规则定义。

## 模板类型

### 1. 简单规则模板（simple-rule-template.md）

**适用场景**：
- 输入字段明确、数量有限（≤10个）
- 判断逻辑可用数学公式或布尔表达式表达
- 不需要跨文档推理
- 不需要语义理解

**典型示例**：
- 数值范围校验（容量、距离、面积）
- 简单计算校验（功率、电流、负荷）
- 枚举值匹配（设备型号、材料规格）
- 单文档字段一致性

**执行方式**：
- `complexity: simple`
- `execution_mode: structured`
- 编译为结构化 JSON，由规则引擎快速执行

### 2. 复杂规则模板（complex-rule-template.md）

**适用场景**：
- 需要多模态文档解析（PDF + Excel + 图纸）
- 需要跨文档实体对齐
- 需要语义理解和推理
- 输入字段不固定或需要动态提取
- 需要专家经验判断

**典型示例**：
- 跨文档一致性校验
- 设计方案合理性评估
- 图纸与清册对比
- 专家意见匹配

**执行方式**：
- `complexity: complex`
- `execution_mode: llm_reasoning`
- 保留 Markdown 引用，由 LLM Agent 推理执行

## 如何选择模板

使用以下决策树选择合适的模板：

```
是否需要跨文档推理？
├─ 是 → 复杂规则模板
└─ 否
    └─ 判断逻辑是否可用公式表达？
        ├─ 是 → 简单规则模板
        └─ 否 → 复杂规则模板

是否需要语义理解？
├─ 是 → 复杂规则模板
└─ 否 → 简单规则模板

输入字段是否明确且有限？
├─ 是 → 简单规则模板
└─ 否 → 复杂规则模板
```

## 模板字段说明

### Frontmatter 字段

| 字段 | 说明 | 必填 |
|------|------|------|
| `rule_id` | 规则唯一标识，格式：R-{专题}-{类别}-{序号} | ✓ |
| `rule_name` | 规则名称，简短描述 | ✓ |
| `version` | 规则版本号，格式：major.minor.patch | ✓ |
| `category` | 规则分类，格式：专题/子类别 | ✓ |
| `complexity` | 规则复杂度：simple \| complex | ✓ |
| `execution_mode` | 执行模式：structured \| llm_reasoning | ✓ |
| `status` | 规则状态：active \| draft \| deprecated | ✓ |
| `created_at` | 创建日期，格式：YYYY-MM-DD | ✓ |
| `updated_at` | 更新日期，格式：YYYY-MM-DD | ✓ |
| `author` | 作者或编写组 | ✓ |
| `tags` | 标签数组，用于检索和分类 | ✓ |

### 规则 ID 命名规范

格式：`R-{专题代码}-{类别代码}-{序号}`

**专题代码**：
- `KGZ`：开关站
- `JJ`：技经资料
- `JG`：竣工资料

**类别代码**：
- `EQ`：设备选型
- `SF`：安全距离
- `CS`：造价核查
- `FE`：字段抽取
- `CC`：一致性校验

**示例**：
- `R-KGZ-EQ-001`：开关站 > 设备选型 > 第1条规则
- `R-JJ-CC-005`：技经资料 > 一致性校验 > 第5条规则

## 编写流程

1. **选择模板**：根据规则特点选择简单或复杂模板
2. **复制模板**：将模板复制到对应的分类目录
3. **填写 frontmatter**：按照字段说明填写元数据
4. **编写规则内容**：
   - 规则描述：说明检查目的
   - 适用范围：明确适用场景
   - 规范依据：列出相关规范条款
   - 检查条件：定义输入和判断逻辑
   - 结果模板：定义输出格式
   - 例外情况：说明豁免条件
   - 测试用例：提供验证示例
5. **更新索引**：在 `source/index.json` 中添加规则记录

## 编译流程

规则编写完成后，需要编译为 JSON 格式供执行引擎使用：

### 简单规则编译

Markdown 源文件 → 结构化 JSON：
- 提取 frontmatter 为 JSON 元数据
- 解析检查条件为 `check_logic` 对象
- 解析结果模板为 `result_templates` 对象
- 生成 `compiled/{category}/{rule_id}.json`

### 复杂规则编译

Markdown 源文件 → JSON 引用：
- 提取 frontmatter 为 JSON 元数据
- 保留 `rule_file` 字段指向 Markdown 源文件
- 生成 `compiled/{category}/{rule_id}.json`

**注意**：首期编译流程为手工执行，后续可开发自动化编译工具。

## 版本管理

规则修订时需要：
1. 更新 `version` 字段（遵循语义化版本）
2. 更新 `updated_at` 字段
3. 在"变更历史"章节添加变更记录
4. 如果是重大变更，将旧版本归档到 `versions/` 目录
