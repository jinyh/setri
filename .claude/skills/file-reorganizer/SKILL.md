# File-Reorganizer 项目文件标准化整理 Skill

## 触发条件

当用户提到以下关键词时激活本 Skill：
- 文件整理、文件归档、文件重组、reorganize files
- 标准化目录、存储规范
- file-reorganizer

## 概述

本 Skill 将 `项目测试数据/` 中散落的原始项目文件，按新的存储规范整理到标准化目录结构 `data/projects/{project_id}/`。

**核心设计原则**：
- **中文命名**：所有目录和文件使用中文命名
- **两层架构**：过程管理（Phase 2 引入）+ 设计文件（Phase 1）
- **项目级平铺**：项目级文件直接放在 `项目级/` 目录下，不再细分子目录
- **分项结构统一**：每个分项（开关站、街坊站-1等）都有完全相同的 4 个子目录
- **说明书归并**：各专业子项下的"说明书"目录内容去重后归入 `项目级/可研报告.pdf`
- **其他依据独立**：保持为 `项目级/其他依据/` 独立目录

**核心原则**：**Claude 负责语义匹配（目录名 → 标准分类），Bash cp 负责文件复制**。
- 不写 Python 脚本——核心难点是语义匹配，这是 Claude 的强项
- 复制而非移动，保留源文件不动
- 仅处理常见文件类型：pdf, xlsx, xls, docx, doc, pptx
- 跳过：.ofd, .bdd3, .bpz17, .DS_Store, .db 等

## Phase 0: 参数确认

与用户确认以下参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| source_dir | `项目测试数据/` | 原始数据根目录 |
| target_dir | `data/projects/` | 标准化输出根目录 |
| projects | 全部 3 个 | 要处理的项目（可选单个） |

确认后进入 Phase 1。

## Phase 1: 扫描 & 语义映射

对每个项目，执行以下步骤：

### 1a. 定位源目录

根据项目注册表，构造源路径：
- 成果资料根：`{source_dir}/修改前/成果资料({full_name}（{project_id}）)/成果资料/`
- 送审版根：`{source_dir}/修改前/成果资料({full_name}（{project_id}）)/成果资料/送审版/`
- 审定版根：`{source_dir}/修改后/成果资料({full_name}（{project_id}）)/成果资料/审定版/`

用 `ls` 列出目录的一级子目录。

### 1b. 映射过程管理文件

扫描成果资料根目录（不含"送审版"和"审定版"本身），将每个一级子目录映射到过程管理目录：

**过程管理文件识别**：
- 关键节点时间 → 过程管理/关键节点时间/
- 项目概况信息 → 过程管理/项目概况信息/
- 设计人员信息 → 过程管理/设计人员信息/
- 预审专家意见 → 过程管理/预审专家意见/
- 正式评审专家意见 → 过程管理/正式评审专家意见/
- 收口评审专家意见 → 过程管理/收口评审专家意见/
- 评审意见 → 过程管理/评审意见/
- 发文意见、发文意见（WORD版） → 过程管理/发文意见/（合并）
- 项目评审记录 → 过程管理/项目评审记录/
- 评分文件 → 过程管理/评分文件/

**注意**：过程管理文件不区分送审版/审定版，从修改前和修改后的成果资料根目录中提取，合并到同一目标目录。

### 1c. 映射项目级文件

扫描 `送审版/` 下的一级子目录：

1. **项目级文件识别**：短名称目录（不含项目全名前缀）→ 匹配项目级文件同义词表
2. **专业子项识别**：长名称目录（含项目全名前缀）→ 提取括号/空格后的专业关键词 → 匹配专业类型

**项目级文件处理**：
- 设计委托书、中标通知书、供电方案审核单、财务审核表、内审意见、投资汇总表、主要设备材料清册 → 直接复制到 `项目级/`
- 其他依据 → 保持为 `项目级/其他依据/` 独立目录，内部文件保持原始结构

### 1d. 映射专业子项

对每个识别出的专业子项目录：

1. **专业类型识别**：从长命名目录中提取专业关键词
   - 提取括号内容：`...工程（KT站）` → `KT站`
   - 提取空格后内容：`...工程 内线电缆-0.4kV` → `内线电缆-0.4kV`
   - 匹配到标准分项名：开关站、街坊站、站内电缆、高压电缆、低压电缆、通信、信息采集

2. **街坊站编号处理**：
   - 多个街坊站按目录名排序
   - 分配编号：街坊站-1, 街坊站-2, ...

3. **分项子目录分类**：
   - 工程图纸：其他图纸、设计图纸、图纸资料、电缆通道断面图
   - 建筑图纸：土建总平面及竖向布置图、土建总平面图、竖向布置图
   - 工程估算书：工程估算书、工程概算书（仅 PDF 版，排除软件版）
   - 材料清册：主要建设材料表、主要材料清册、设备材料清册、材料表、拆除设备清单、拆除设备材料清单

4. **说明书归并**：
   - 查找各专业子项内的"说明书"、"可研报告"、"初设报告"、"设计报告"子目录
   - 收集所有 PDF 文件
   - 去重（同名文件只保留一份）
   - 归并到 `项目级/可研报告/`

### 1e. 映射审定版内部

同 1c-1d 逻辑，扫描 `审定版/` 下的目录。
- 项目级文件：与送审版独立，分别映射到 `设计文件/审定版/项目级/`
- 专业子项：独立映射到 `设计文件/审定版/` 下

### 1f. 生成映射计划

输出结构化映射表，格式：

```
映射计划 — {short_name} ({project_id})
============================================
| # | 源路径 | → | 目标路径 | 文件数 | 备注 |
|---|--------|---|---------|--------|------|
| 1 | .../送审版/设计中标通知书或委托书/ | → | 设计文件/送审版/项目级/ | 3 | |
| 2 | .../送审版/...（KT站）/其他图纸/ | → | 设计文件/送审版/开关站/工程图纸/ | 5 | |
| ... | | | | | |
```

标注：
- `[未匹配]`：无法映射的目录
- `[歧义]`：可能匹配多个分类的目录
- `[跳过]`：仅含排除文件类型的目录

## Phase 2: 用户审核映射计划（可选）

将 Phase 1 的映射表展示给用户，逐项目确认：

1. 展示每个项目的映射表（Phase 1e 格式）
2. 高亮标注 `[未匹配]`、`[歧义]`、`[跳过]` 的条目
3. 统计摘要：已匹配 N 个目录，未匹配 M 个，跳过 K 个
4. 询问用户是否需要调整映射

用户确认后进入 Phase 3。如有调整，修改映射表后重新确认。

## Phase 3: 执行文件复制

### 3a. 创建目录结构

对每个项目，用 `mkdir -p` 创建完整目录树：

```bash
PROJECT_DIR="{target_dir}/{project_id}"

# 根目录
mkdir -p "$PROJECT_DIR"

# 过程管理目录（仅创建有内容的）
mkdir -p "$PROJECT_DIR/过程管理/关键节点时间"
mkdir -p "$PROJECT_DIR/过程管理/项目概况信息"
mkdir -p "$PROJECT_DIR/过程管理/设计人员信息"
mkdir -p "$PROJECT_DIR/过程管理/预审专家意见"
mkdir -p "$PROJECT_DIR/过程管理/正式评审专家意见"
mkdir -p "$PROJECT_DIR/过程管理/收口评审专家意见"
mkdir -p "$PROJECT_DIR/过程管理/评审意见"
mkdir -p "$PROJECT_DIR/过程管理/发文意见"
mkdir -p "$PROJECT_DIR/过程管理/项目评审记录"
mkdir -p "$PROJECT_DIR/过程管理/评分文件"

# 设计文件目录
mkdir -p "$PROJECT_DIR/设计文件"/{送审版,审定版}

# 项目级目录
mkdir -p "$PROJECT_DIR/设计文件/送审版/项目级"/{可研报告,其他依据}
mkdir -p "$PROJECT_DIR/设计文件/审定版/项目级"/{可研报告,其他依据}

# 分项目录（仅创建有内容的）
# 示例：开关站
mkdir -p "$PROJECT_DIR/设计文件/送审版/开关站"/{工程图纸,建筑图纸,工程估算书,材料清册}
mkdir -p "$PROJECT_DIR/设计文件/审定版/开关站"/{工程图纸,建筑图纸,工程估算书,材料清册}

# 街坊站示例（多个编号）
mkdir -p "$PROJECT_DIR/设计文件/送审版/街坊站-1"/{工程图纸,建筑图纸,工程估算书,材料清册}
mkdir -p "$PROJECT_DIR/设计文件/送审版/街坊站-2"/{工程图纸,建筑图纸,工程估算书,材料清册}

# ... 按映射表创建其他目录
```

### 3b. 按映射表复制文件

遍历映射表，对每条映射执行：

```bash
# 仅复制常见文件类型
find "{源路径}" -maxdepth 1 -type f \( \
  -iname "*.pdf" -o -iname "*.xlsx" -o -iname "*.xls" \
  -o -iname "*.docx" -o -iname "*.doc" -o -iname "*.pptx" -o -iname "*.ofd" \
\) -exec cp {} "{目标路径}/" \;
```

**特殊处理**：

1. **过程管理文件**：
   - 从修改前和修改后的成果资料根目录提取
   - 合并到同一目标目录（不区分送审版/审定版）
   - 发文意见和发文意见（WORD版）合并到同一目录

2. **项目级文件**：
   - 设计委托书、中标通知书等 → 直接复制到 `项目级/`
   - 其他依据 → 保持为 `项目级/其他依据/` 独立目录

3. **可研报告归并**：
   - 从各专业子项的"说明书"目录收集 PDF 文件
   - 按文件名去重
   - 复制到 `项目级/可研报告/`

4. **分项子目录**：
   - 工程图纸：复制图纸类子目录中的文件
   - 建筑图纸：复制土建总平面及竖向布置图中的文件
   - 工程估算书：复制工程估算书（PDF 版）中的文件，排除软件版
   - 材料清册：复制材料清册、拆除设备清单中的文件

### 3c. 执行顺序

按项目逐个执行，每个项目内按以下顺序：
1. mkdir -p 创建目录
2. 复制过程管理文件（从修改前和修改后合并）
3. 复制项目级文件（送审版和审定版独立）
4. 复制可研报告（含说明书归并）
5. 复制分项子目录文件（送审版 + 审定版）

每个项目完成后输出简要进度。

## 已知限制

1. **语义匹配非 100%**：极端命名变体可能需要人工干预（Phase 2 审核环节兜底）
2. **文件去重**：可研报告去重基于文件名，不做内容级比对
3. **街坊站排序**：多个街坊站的编号顺序基于目录名排序，可能与实际编号不完全一致
4. **说明书内容判断**：说明书目录中绝大多数为可研报告，但个别可能混入其他文件（如情况说明），需在 Phase 2 审核时关注
