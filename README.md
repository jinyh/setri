# Setri — 配电网业扩智能评审副驾驶 (Co-Reviewer)

> 统一交付演示系统 — 经研院项目群智能审查/核查工作台

## 项目概述

为上海经研院配电网评审专家构建的智能辅助评审系统，通过建设统一交付演示系统，承接 `合同/` 目录下 8 份项目材料的研究成果，形成可展示、可讲解、可扩展、可验收的演示平台。

系统采用"公共能力底座 + 核心主线模块 + 专题扩展模块"的产品结构：

- **公共能力底座**：项目与案例管理、多模态资料接入与解析、知识与规则中心、核查与推理引擎、AI 助手与问答、结果输出
- **核心主线模块**：M1 业扩项目智能评审、M2 技经资料预审与评审
- **专题扩展模块**：M3 开关站专家知识库、M4 工程智慧进度管控、M5 竣工资料智能核查

## 统一业务主线

所有模块复用同一条业务主线：

1. 选择项目或案例
2. 导入资料或过程数据（首期以预注册样例项目选择为主）
3. 进行解析、抽取和结构化
4. 调用知识库、规则库或图谱进行匹配
5. 执行核查、推理或预警分析
6. 展示问题、证据与建议
7. 生成人工可确认的最终结论或报告

## 技术栈

- 前端：Next.js（App Router）+ TypeScript + shadcn/ui + Tailwind CSS
- 后端 API：FastAPI
- 后台任务：Celery + Redis
- 存储：文件系统 + JSON 元数据优先，数据库位预留
- VLM：Gemini 3.0 Flash（MVP）→ Qwen VL（微调私有化）
- 部署：Docker 化作为后续交付增强项

## 设计原则

- **三大文档协同**：PRD 负责产品定义，重构计划负责工程实现，Milestone 计划负责交付节奏
- **先统一，再分专题**：所有模块共享同一套主线，不出现多套割裂流程
- **先主线闭环，再专题增强**：首期核心主线（M1/M2）真实可运行，M3-M5 仅骨架
- **证据链优先**：所有自动结论必须可回溯到资料片段、规范条款或规则依据
- **人机协同**：输出结论支持人工复核，不以"全自动替代专家"为目标
- **首期以预注册样例驱动**：不把浏览器内新增真实资料包上传作为硬验收前提

## 术语表

| 术语 | 含义 |
|------|------|
| 送审版 | 专家收到的待审材料（修改前），系统输入基准 |
| 审定版 | 专家评审后的修正版本（修改后），参考标准答案 |
| 三元关联 | 专家意见 ↔ 设计文件 ↔ 引用规范 的关联结构 |
| 管道站 | 数据管道中的一个处理阶段（P1-P5） |
| 交叉提炼 | "应然"（规范）× "实然"（专家）→ 可执行规则 |
| 智能体协同 | 多智能体系统（MAS）架构，信息抽取/审查/意见落实/总控智能体分工协作 |
| RAG | 检索增强生成，用于知识增强的智能问答 |
| 证据链 | 核查结论回溯到 file + page + snippet 的可解释输出 |
| 预注册样例 | 平台预先录入的样例项目/案例，用于演示驱动 |
