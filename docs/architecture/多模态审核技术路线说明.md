## 1. 目标与边界

本文件用于定义统一交付演示系统在多模态审核场景下的技术主路线，重点回答三类问题：

- 不同资料类型优先采用什么解析路径
- 模型、规则引擎和确定性工具各自负责什么
- 自动判定能力如何在“证据可追溯”的前提下逐步提高

本文件不承诺具体厂商或模型名称。模型选型以能力层表达，避免 PRD 与实施长期绑定在单一供应商命名上。

## 2. 总体原则

系统统一遵守以下原则：

- 证据优先于结论。所有问题结论都必须可回溯到资料片段、规范条款或规则依据。
- 结构化优先于端到端。能用结构化解析、模板识别或确定性提取完成的任务，不默认交给 VLM。
- 自动化深度服从可解释性。先保证“可复验”，再追求“更少人工”。
- 模型不做确定性判定。模型负责感知、候选抽取、编排和表述，最终阈值比较、逻辑校验和规则结论由工具侧输出。
- 关键字段必须带来源和置信度。关键数字、关键文本和定位信息缺少来源追踪时，不得直接进入自动判定链。

## 3. 分层解析路线

### 3.1 统一解析分层

多模态审核采用四层解析路线：

1. 文件识别与归档层：识别文件类型、目录语义、版本和来源路径。
2. 结构化抽取层：优先使用表格解析、版面解析、模板抽取和文本切分。
3. 视觉理解补盲层：仅在复杂版面、扫描件、图纸标注和跨区域定位场景下使用 VLM。
4. 规则执行与结果生成层：将抽取结果输入规则、工具和结果模板，输出可追溯结论。

### 3.2 按资料类型的优先路径

| 资料类型 | 主解析路径 | VLM 角色 | 默认输出 |
|----------|------------|----------|----------|
| Excel/结构化表格 | 表格解析、字段映射、模板校验 | 仅补充表头歧义和异常单元格理解 | 字段、表格记录、校验结果 |
| 可检索 PDF 文本 | 版面切分、章节识别、规则化抽取 | 仅处理复杂跨段定位和低质量排版 | 章节、字段、候选证据片段 |
| 扫描 PDF/图片 | OCR + 版面分析 + VLM 复核 | 负责复杂版面理解和低质量文本补盲 | 文字区域、字段、证据锚点 |
| 图纸/示意图 | 图像切片、目标区域识别、VLM/视觉模型抽取 | 负责尺寸、标注、设备关系理解 | 尺寸、对象、定位区域、置信度 |
| 混合型文档 | 分页分区路由到上述路径 | 负责跨模态整合 | 统一结构化结果 |

### 3.3 三类输出级别

| 级别 | 说明 | 适用条件 |
|------|------|----------|
| 自动判定 | 可直接产出规则结论 | 规则可确定执行，证据链完整，关键值置信度达标 |
| 半自动提示 | 生成待确认问题 | 规则部分成立，但关键字段或证据定位仍需人工复核 |
| 仅人工参考 | 只提供候选线索 | 规则不可确定执行，或问题依赖主观判断 |

## 4. 模型、工具与规则边界

### 4.1 模型职责

- 文档/图纸感知
- 候选字段抽取
- 资料片段定位
- 多步执行编排
- 面向用户的结果表述

### 4.2 工具职责

- 算术计算
- 阈值比较
- 一致性校验
- 规则条件求值
- 最终合规结论输出

### 4.3 规则职责

- 定义检查对象
- 定义所需字段
- 定义执行工具和判断条件
- 定义严重级别和结果模板
- 定义所需证据类型

## 5. 执行 trace 与证据要求

所有进入 P4/P5 的结果必须保留最小执行 trace：

- 输入资料引用
- 抽取字段和值
- 抽取来源页码或区域锚点
- 置信度
- 命中规则与规则版本
- 调用的工具及其返回结果
- 最终输出的结论项

证据定位最低要求为 `file + page + snippet`。图纸和复杂版面场景应进一步保留区域锚点信息，便于复核。

## 6. 模型部署与数据治理

模型路线按能力层分两段推进：

- 外部验证模型层：用于快速验证复杂文档理解能力，积累提示词模板、错误样本和评测数据。
- 内网私有模型层：在数据治理条件满足后承接长期运行，降低对外部 API 的依赖。

涉及真实项目资料时，必须明确：

- 哪些文件或字段允许出网
- 脱敏规则由谁维护
- 何时切换到内网私有模型层
- 哪些任务只能在内网执行

数据治理规则未明确前，自动化路线不得默认以“外部 API 全量处理真实资料”为前提。

## 7. 与 P3-P5 的关系

- P2 输出结构化结果和证据锚点，为 P3 和 P5 提供共同底座。
- P3 消费 P2 结果做知识抽取和反馈沉淀，但不替代 P5 的主判定链。
- P4 将规范条款、设计文件命中结果和专家反馈转为可执行规则。
- P5 以“规则库直接评审设计文件”为主链路，模型用于感知与解释，不直接产出最终合规裁决。

## 8. 验收关注点

- 不同资料类型是否走了正确的主解析路径
- 关键字段是否保留来源与置信度
- 自动判定结论是否可追溯到规则与证据
- 模型调整后是否能通过回归验证
- 外部验证模型层与内网私有模型层是否具备可替换性
