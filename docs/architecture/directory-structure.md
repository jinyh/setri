## 一、评审流程时间轴（来源：关键节点时间.xls）

```
项目创建 → 计划申报 → 计划审核通过 → 项目发布
  → 送审资料上传 → 送审申报
    → 预评审（技术+技经多轮） → 预审通过
      → 正式会议开展
        → 收口资料上传 → 收口申报 → 上传评审意见
          → 收口资料审核（技术+技经多轮） → 收口审核结束
            → 结束评分 → 评审意见结束编制 → 发文
```

## 五、关键差异与问题汇总

### 5.1 原分析文件的主要问题

原 `test-data-diff-analysis.json` 将「送审版→审定版」视为目录重命名，实为两个独立版本目录，
应并列存在于合并视图中，而非替换关系。

### 5.2 跨项目命名不统一（需模糊匹配）

| 问题类型 | 示例 |
|---------|------|
| 分隔符不统一 | 项目1/2用空格，项目3用括号 |
| 同义词混用 | 信息采集 / 用电信息采集；内线通信 / 通信 |
| 术语差异 | 设计中标通知书或委托书 / 设计委托书 |
| 辅助文件差异 | 内审意见 / 内审意见及签到单；其它依据 / 其他依据 / 其他 |
| 括号全半角混用 | (PDF版) vs （PDF版） |
| 开关站命名体系 | 项目1用缩写（KT/PT），项目2/3用全称 |

### 5.3 可选件目录（非三项目共有）

- `供电方案审核单`：仅项目3
- `通信方案确认单`：仅项目3
- `财务审核表`：仅项目1
- `主要设备材料清册`：仅项目3
- `评分文件`（根目录）：仅项目2、3
- `案例分析报告`（审定版内）：项目1、3有，项目2无
