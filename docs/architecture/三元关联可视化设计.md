## 1. 设计背景

### 1.1 三元关联定义

三元关联是专家知识库（P3）的核心资产，表示：
- **专家意见**：专家在评审过程中提出的问题或建议
- **设计文件**：意见所指向的具体设计文件位置
- **引用规范**：意见所依据的技术规范条款

### 1.2 使用场景

系统需要支持四类主要场景：
1. **模式发现**（优先级最高）：横向对比多个项目，发现相似问题模式
2. **定位问题**（优先级第二）：快速定位意见指向的文件位置和证据片段
3. **追溯依据**（优先级第三）：查看意见背后的规范条款依据
4. **演化追踪**（优先级第四）：追踪意见如何转化为可执行规则

### 1.3 组织维度

用户希望按以下维度组织三元关联：
- **主维度**：文件类型（工程估算书、单线图、材料清册等）
- **次维度**：问题类型（负荷计算错误、设备选型不当等）
- **层次关系**：先按文件类型分组，再在每个文件类型内按问题类型细分

## 2. 整体架构

### 2.1 布局方案

采用**三区域布局**：

```
┌─────────────────────────────────────────────────────────────┐
│  顶部：文件类型 Tab + 问题类型筛选 + 搜索/排序控制           │
├──────────────────────────────────┬──────────────────────────┤
│                                  │                          │
│  主区域：可展开表格              │  右侧：模式洞察面板      │
│  - 默认显示摘要信息              │  - 高频规范              │
│  - 点击展开完整三元关联          │  - 高频文件位置          │
│  - 支持多选、批量操作            │  - 问题分布统计          │
│                                  │  - 相似问题聚类          │
│                                  │                          │
└──────────────────────────────────┴──────────────────────────┘
```

### 2.2 设计理念

- **信息密度优先**：表格形式适合处理大量数据，支持快速浏览和筛选
- **渐进式展开**：默认显示摘要，按需展开详情，避免信息过载
- **自动化洞察**：右侧面板自动聚合统计数据，无需手动对比即可发现模式
- **操作效率**：支持批量操作，适合专家快速审查和处理

## 3. 主表格设计

### 3.1 默认列（折叠状态）

| 列名 | 宽度 | 说明 |
|------|------|------|
| 选择框 | 40px | 支持多选，用于批量操作 |
| 项目名称 | 120px | 可点击跳转到项目详情页 |
| 意见摘要 | 300px | 截取前50字，hover显示完整文本 |
| 文件类型 | 100px | 带图标（如📄工程估算书、📐单线图） |
| 问题类型 | 120px | 带颜色标签（如🔴严重、🟡警告） |
| 置信度 | 80px | 进度条样式，范围 0.0-1.0 |
| 审查状态 | 100px | 待审查/已确认/已忽略/已转规则 |
| 操作 | 80px | 展开/收起按钮 |

### 3.2 展开行（显示完整三元关联）

展开后在当前行下方插入详情区域，分为三列：

```
┌─────────────────┬─────────────────┬─────────────────┐
│  专家意见       │  设计文件定位   │  引用规范       │
│  - 完整文本     │  - 文件路径     │  - 规范名称     │
│  - 来源阶段     │  - 页码/位置    │  - 条款编号     │
│  - 提出专家     │  - 文件预览     │  - 条款内容     │
│  - 提出时间     │  - 证据片段     │  - 查看原文链接 │
│                 │                 │                 │
│  [查看相似问题] │  [跳转到文件]   │  [查看规范全文] │
└─────────────────┴─────────────────┴─────────────────┘
```

**展开区域特点：**
- 三列等宽，清晰展示三元关联的三个要素
- 每列底部提供操作按钮，支持深入查看或跳转
- 文件预览支持缩略图（图纸类）或文本片段（文档类）
- 证据片段高亮显示问题区域

### 3.3 表格交互

- **排序**：支持按置信度、时间、项目名称排序
- **筛选**：支持按文件类型、问题类型、置信度范围、审查状态筛选
- **搜索**：全文搜索意见文本、规范名称、项目名称
- **多选**：支持 Shift 连选、Ctrl/Cmd 多选
- **虚拟滚动**：支持大数据集（1000+ 条记录）流畅滚动

## 6. 交互流程设计

### 6.1 场景1：模式发现（优先级最高）

**用户目标**：发现跨项目的相似问题模式

**操作流程**：
1. 进入页面 → 默认显示"工程估算书"Tab（假设这是问题最多的文件类型）
2. 右侧"相似问题聚类"卡片自动显示该文件类型下的问题模式
3. 点击某个模式（如"负荷计算系数错误"）→ 主表格自动筛选出相关记录
4. 表格中相关记录高亮显示，可批量展开查看详情
5. 点击"查看相似问题"按钮 → 跨文件类型显示所有相似问题

**关键设计**：
- 模式聚类自动化，无需手动对比
- 一键筛选相关记录
- 支持跨文件类型查看

### 6.2 场景2：定位问题（优先级第二）

**用户目标**：快速定位意见指向的文件位置

**操作流程**：
1. 在表格中点击某条记录的"展开"按钮
2. 展开区域的"设计文件定位"列显示：
   - 文件路径（可点击）
   - 页码/位置
   - 文件预览（缩略图或文本片段）
   - 证据片段（高亮显示问题区域）
3. 点击"跳转到文件"→ 打开文件查看器，自动定位到问题位置

**关键设计**：
- 文件预览直接嵌入展开区域
- 证据片段高亮显示
- 一键跳转到文件详情页

### 6.3 场景3：追溯依据（优先级第三）

**用户目标**：查看意见背后的规范条款依据

**操作流程**：
1. 在展开的详情区域，查看"引用规范"列
2. 显示规范名称、条款编号、条款内容摘要
3. 点击"查看规范全文"→ 打开规范浏览器，定位到具体条款
4. 规范浏览器中高亮显示相关条款，并显示"引用该条款的其他意见"

**关键设计**：
- 条款内容摘要直接显示
- 规范浏览器支持条款定位
- 显示条款的其他引用情况

### 6.4 场景4：演化追踪（优先级第四）

**用户目标**：追踪意见如何转化为可执行规则

**操作流程**：
1. 在表格的"审查状态"列，筛选"已转规则"状态
2. 展开记录后，显示"规则演化链路"：
   ```
   专家意见 → 知识模式 → 规则候选 → 正式规则
   (2024-03)  (2024-05)   (2024-07)   (2024-09)
   ```
3. 点击规则ID → 跳转到规则详情页
4. 规则详情页显示：
   - 规则内容
   - 来源的三元关联记录（可能多条）
   - 规则修订历史
   - 规则命中统计

**关键设计**：
- 演化链路时间线可视化
- 规则与三元关联双向关联
- 规则详情页完整展示演化历史

## 8. 响应式与性能优化

### 8.1 数据加载策略

**分页加载：**
- 主表格默认显示 50 条记录
- 支持虚拟滚动（处理大数据集）
- 展开详情时才加载文件预览和证据片段

**缓存策略：**
- 文件类型切换时缓存已加载的数据
- 模式洞察面板的统计数据预计算并缓存
- 文件预览图片使用 CDN 缓存

**实时更新：**
- 当用户在其他页面修改三元关联状态时，当前页面自动刷新
- 模式洞察面板的统计数据实时重新计算

### 8.2 响应式设计

- **大屏（≥1920px）：** 三列布局（表格 + 模式洞察面板）
- **中屏（1280-1919px）：** 模式洞察面板可折叠，默认展开
- **小屏（<1280px）：** 模式洞察面板默认折叠，表格占满宽度

### 8.3 性能指标

- 首屏加载时间 < 2s
- 表格展开/收起响应时间 < 300ms
- 筛选操作响应时间 < 500ms
- 支持同时展开最多 10 条记录

### 9.4 数据库设计（核心表）

```sql
-- 三元关联表
CREATE TABLE triad_relations (
    id UUID PRIMARY KEY,
    project_id VARCHAR(50),
    opinion_id VARCHAR(50),
    opinion_text TEXT,
    source_stage VARCHAR(20),
    file_type VARCHAR(50),
    issue_type VARCHAR(50),
    document_ref JSONB,
    evidence_ref JSONB,
    clause_ref JSONB,
    rule_ref VARCHAR(50),
    confidence DECIMAL(3,2),
    review_status VARCHAR(20),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 索引
CREATE INDEX idx_file_type ON triad_relations(file_type);
CREATE INDEX idx_issue_type ON triad_relations(issue_type);
CREATE INDEX idx_confidence ON triad_relations(confidence);
CREATE INDEX idx_status ON triad_relations(review_status);
```

### 9.5 关键实现细节

**虚拟滚动实现：**
- 使用 TanStack Table 的 `useVirtualizer` hook
- 只渲染可见区域的行，提升大数据集性能

**模式聚类算法：**
- 使用后端 NLP 模型对 `opinion_text` 进行语义相似度计算
- 结合 `clause_ref` 共现分析，提高聚类准确度
- 预计算并缓存聚类结果，前端直接展示
