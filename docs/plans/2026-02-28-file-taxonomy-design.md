# 设计方案：项目文件分类体系 + PRD 疏漏修补

> 日期：2026-02-28 | 关联 PRD 版本：v0.2 → v0.5

## Context

`directory-structure.md` 揭示了实际项目数据的两层结构（成果资料根目录 13 个过程管理目录 + 送审版/审定版内部设计文件），
但当前 PRD v0.2 仅覆盖了第二层（送审版内部的 F0-F10 共 11 类文件），完全遗漏了第一层过程管理数据。

用户决策：
- 输入范围：**分阶段扩展**（Phase 1 送审版设计文件，Phase 2 引入过程数据）
- 目录结构：**逻辑分类 + 物理存储**
- 分类方案：**三层分组**（过程管理 + 辅助文件 + 专业图纸）

## 二、文件分类体系（逻辑模型）

### 2.1 三层分类结构

```
项目文件
├── 过程管理类 (P-xx)     ← Phase 2 引入
│   ├── P-01 关键节点时间
│   ├── P-02 项目概况信息
│   ├── P-03 设计人员信息
│   ├── P-04 预审专家意见
│   ├── P-05 正式评审专家意见
│   ├── P-06 评分文件
│   ├── P-07 收口评审专家意见
│   ├── P-08 评审意见
│   ├── P-09 发文意见
│   └── P-10 项目评审记录
│
└── 设计文件类
    ├── 共享辅助文件 (A-01~A-09)  ← 不区分送审/审定版
    │   ├── A-01 设计委托书          [必备]
    │   ├── A-02 内审意见            [必备]
    │   ├── A-03 其他依据            [必备]
    │   ├── A-04 投资汇总表          [必备]
    │   ├── A-05 财务审核表          [可选]
    │   ├── A-06 供电方案审核单       [可选]
    │   ├── A-07 通信方案确认单       [可选]
    │   ├── A-08 主要设备材料清册     [必备]
    │   └── A-09 案例分析报告        [可选]
    │
    ├── 送审版 (submitted/)
    │   ├── 可研报告                  [必备, 版本敏感, A-10]
    │   ├── 专业图纸 (D-xx)          ← 每项含 wiring(电气接线图) + layout(平面布置图) 两个逻辑子类型
    │   │   ├── D-01 开关站电气一次
    │   │   ├── D-02 街坊站电气一次
    │   │   ├── D-03 站内电缆
    │   │   ├── D-04 高压电缆(10kV)
    │   │   ├── D-05 低压电缆(0.4kV)
    │   │   ├── D-06 通信
    │   │   └── D-07 信息采集
    │   └── 工程估算 (E-xx)          ← 与 D-xx 一一对应
    │       ├── E-01 开关站工程估算
    │       ├── E-02 街坊站工程估算
    │       ├── E-03 站内电缆工程估算
    │       ├── E-04 高压电缆工程估算
    │       ├── E-05 低压电缆工程估算
    │       ├── E-06 通信工程估算
    │       └── E-07 信息采集工程估算
    │
    └── 审定版 (approved/)           ← 结构同送审版
        ├── 可研报告
        ├── drawings/
        └── estimates/
```

> **图纸逻辑子类型说明**：每个 D-xx 在逻辑上包含"电气接线图 (wiring)"和"平面布置图 (layout)"两个子类型。系统通过文件名关键词识别（总平图/平面图/布置图 → layout；接线图/电气/系统图 → wiring），无法区分时标记为"综合图纸 (mixed)"，不报错。不在物理目录中创建子目录。

## 三、物理存储规范

### 3.1 标准化目录模板

```
data/projects/{project_id}/
├── meta.json                          ← 项目元数据
├── original/                          ← 原始文件归档（只读）
│   └── ...
├── process/                           ← 过程管理文件（Phase 2）
│   ├── P-01_关键节点时间/
│   ├── P-04_预审专家意见/
│   └── ...
└── design/                            ← 设计文件（Phase 1）
    ├── auxiliary/                     ← 共享辅助文件（A-01~A-09，不区分版本）
    │   ├── A-01_设计委托书/
    │   ├── A-02_内审意见/
    │   └── ...
    ├── submitted/                     ← 送审版
    │   ├── 可研报告/                 ← 版本敏感辅助文件（A-10）
    │   ├── drawings/                  ← 专业图纸
    │   │   ├── D-01_开关站/
    │   │   ├── D-02_街坊站_01/
    │   │   ├── D-02_街坊站_02/
    │   │   ├── D-03_站内电缆/
    │   │   ├── D-04_高压电缆/
    │   │   ├── D-05_低压电缆/
    │   │   ├── D-06_通信/
    │   │   └── D-07_信息采集/
    │   └── estimates/                 ← 工程估算
    │       ├── E-01_开关站/
    │       ├── E-02_街坊站_01/
    │       ├── E-03_站内电缆/
    │       └── ...
    └── approved/                      ← 审定版
        ├── 可研报告/
        ├── drawings/
        └── estimates/
```

### 3.2 设计要点

- `original/` 保留原始目录结构不动，用于溯源和调试
- `design/submitted/` 和 `design/approved/` 使用规范化命名（type_id + canonical_name）
- 文件导入时通过同义词表 + 模糊匹配将原始目录映射到规范化路径
- `meta.json` 记录原始路径 → 规范化路径的映射关系

## 四、验证方式

1. 用三个测试项目的实际目录验证分类体系覆盖率：每个原始目录都能映射到某个 type_id
2. 检查 PRD 内部一致性：§4.3.7 的文件类型 ID 与 §4.5 规则库中引用的文件类型对应
3. 检查 §4.3.5 覆盖矩阵与 §4.3.7 注册表的 required 字段一致
