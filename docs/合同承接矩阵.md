## 1. 使用原则

本文件是合同内容到统一系统能力的追踪文档，目的不是复写合同全文，而是回答三个问题：

1. 每份合同的核心研究成果在系统中落到哪里
2. 哪些能力属于首期真实交付，哪些属于后续演进
3. 哪些内容尚未验收，必须在系统中保留明确挂接位置

说明：

- “首期承接”表示纳入当前重构后的系统主线或专题骨架
- “后续承接”表示已预留模块边界、对象或页面位置，但不承诺首期真能力
- 论文、专利、研究报告等科研成果不直接转译为功能页面，仅作为成果来源说明保留

## 4. 当前默认结论

- 当前统一系统首期硬交付仍以 M1/M2 主线和 P1/P2 真实闭环为核心。
- M3-M5 在首期以“有明确承接位置的专题骨架”进入系统，不以假数据冒充已完成能力。
- 5 份未验收材料必须持续保留在本矩阵中，并随着实现推进更新“首期承接方式”和“后续承接方式”。
- 如果 `合同/` 目录文件、状态或名称发生变化，应先更新本矩阵，再调整 `docs/prd.md` 中的引用和范围描述。
