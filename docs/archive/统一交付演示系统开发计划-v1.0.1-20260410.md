## Context

根据《统一交付演示系统需求文档 v0.2》和《Milestone计划 v0.2》，本计划旨在构建一个全新的统一交付演示系统，将 8 份科研项目材料的研究成果归并为一个统一平台。

**核心变化**：这不是对现有 prd.md 的增量开发，而是全新架构重构。现有代码仅作为参考和部分复用。

**目标**：建设一个能够完成"导入、解析、核查、追证、输出"的业务型演示系统，支撑业扩评审、技经评审、开关站知识库、进度管控、竣工核查五大模块。

## 系统架构

### 技术栈

- 前端：React + Next.js (App Router) + TypeScript + shadcn/ui + Tailwind CSS
- 后端：FastAPI + Celery + Redis + PostgreSQL
- VLM：Gemini 3.0 Flash (MVP) → Qwen VL (私有化)
- 认证：JWT
- 状态管理：Zustand（全局）+ TanStack Query（服务端数据缓存）
- 部署：Docker + docker-compose（内网服务器）

### 模块结构

```
统一交付演示系统
├── 公共能力底座
│   ├── A. 项目与案例管理
│   ├── B. 多模态资料接入与解析
│   ├── C. 知识与规则中心（规范库、规则库、知识图谱）
│   ├── D. 核查与推理引擎
│   ├── E. AI 助手与问答
│   └── F. 结果输出（问题清单、证据链、报告）
├── 核心主线模块
│   ├── M1：业扩项目智能评审
│   └── M2：技经资料预审与评审
└── 专题扩展模块
    ├── M3：开关站专家知识库
    ├── M4：工程智慧进度管控
    └── M5：竣工资料智能核查
```

### 统一业务主线（所有模块复用）

1. 选择项目或案例
2. 导入资料或过程数据
3. 进行解析、抽取和结构化
4. 调用知识库、规则库或图谱进行匹配
5. 执行核查、推理或预警分析
6. 展示问题、证据与建议
7. 生成人工可确认的最终结论或报告

## 关键设计决策

### 1. 多模态解析

- MVP：Gemini 3.0 Flash API
- 生产：Qwen VL 私有化部署
- 原则：LLM 不做算术、不做最终合规判定、关键数值设置置信度阈值

### 2. 知识图谱存储

- 方案：PostgreSQL JSON 列 + vis-network 前端可视化
- 理由：数据规模有限（~5000节点），无需 Neo4j
- 后续可迁移到专业图数据库

### 3. 证据链实现

- 模式：EAV + 关系追溯链
- 结构：问题定位 → 规范依据 → 专家经验（三层）

### 4. AI 问答集成

- 方案：RAG（检索增强生成）
- 知识源：规范条款 + 专家模式 + 知识节点
- 要求：每个回答标注来源

## 核心数据模型

### ER 关系（核心实体）

```
Project 1---* Material 1---* ParseResult 1---* ExtractedField
Project 1---* ReviewSession 1---* Opinion *---* Evidence
Regulation 1---* Clause *---* Rule *---* ExpertPattern
Clause 1---* TripleLink *--- ExpertPattern
```

### 关键实体字段

- **Project**: id, name, type, status, source_paths, meta_data
- **Material**: project_id, file_type, version, category, path
- **ParseResult**: material_id, parse_type, data(JSON), confidence
- **Opinion**: session_id, rule_id, severity, text, status, source_trace(JSON)
- **Evidence**: material_id, page_number, snippet, trace_type, trace_source(JSON)
- **Clause**: regulation_id, chapter, text, importance, verifiability
- **Rule**: clause_ids[], strength, check_method, check_parameters, opinion_template
