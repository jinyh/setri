# P1-P3 Core Assets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 基于旧版 `prd-v0.8-20260410-pre-rewrite.md`，优先落地可复用的 `P1` 规范条款库与 `P3` 专家知识库核心资产，并以 `P2-lite` 提供最小文件定位与证据支撑。

**Architecture:** 继续沿用当前仓库的 Python 原型骨架，不重建 monorepo，也不先做完整工作台。`P1` 优先复用现有 `reg-extractor` 脚本和 `src/setri/p1_regulations/*` 包装层；`P3` 新增独立知识库模块，直接消费已有 `data/projects/*` 目录和 `P1` 条款资产；`P2-lite` 仅为三元关联生成 `file_ref`、`evidence_ref` 和最小定位结果。

**Tech Stack:** Python 3.10+, uv, pytest, Typer, FastAPI（最小查询接口）, pdfplumber, 现有 `.claude/skills/reg-extractor` 脚本

---

```python
                "file_ref": relative_design_path(pdf),
```

```python
def build_knowledge_assets(project_ids: list[str], regulation_dir: Path, output_dir: Path) -> dict:
    clauses = load_clause_assets(regulation_dir)
    links = []
    for project_id in project_ids:
        opinions = load_formal_review_opinions(project_id)
        evidence = collect_project_evidence(project_id)
        links.extend(link_opinions(opinions, evidence, clauses))
    patterns = summarize_patterns(links)
    write_json(output_dir / "三元关联" / "opinions.json", links)
    write_json(output_dir / "三元关联" / "patterns.json", patterns)
    return {"triple_links": len(links), "patterns": len(patterns)}
```
