# Setri — 配电网业扩智能评审副驾驶 (Co-Reviewer)

## 项目概述

为上海经研院配电网评审专家构建的智能辅助评审系统，以 10kV 开关站为切入点，覆盖预审—评审—收口全流程。

## 当前阶段

**Phase 1 MVP 搭建** — 聚焦两件事：

1. **PRD 迭代**：`docs/prd.md`（当前 v0.2 草稿），使用 iterative-prd 技能管理版本
2. **数据治理**：基于 3 个测试项目（崇明盛世新苑、奉贤悦景名筑、奉贤咉懿佳园）进行数据分析和规范化

## 关键文件

| 路径 | 说明 |
|------|------|
| `docs/prd.md` | 产品需求文档（迭代中） |
| `docs/architecture/test-data-diff-analysis.json` | 测试数据差异分析结果 |
| `data/regulations/` | 技术规范文件 |
| `技术规范文件/` | 原始技术规范 PDF |
| `项目测试数据/` | 3 个项目的送审版/审定版目录数据 |
| `产品需求文档.docx` | 原始 PRD（已迁移为 Markdown） |

## 工作约定

- PRD 修改必须走 iterative-prd 流程：更新内容 → 升版 → 追加变更记录 → git commit
- PRD 版本格式：v0.x（草稿），v1.0（正式版，需用户明确确认）
- 数据分析结论写入 `docs/architecture/` 目录，JSON 格式
- 系统输入以"送审版"为基准（详见 PRD 4.3.4）
- 文件类型识别需支持模糊匹配，容忍命名不一致（详见 PRD 4.3.6）

## 技术栈（规划）

- 后端：Python (FastAPI)
- 前端：Vue 3 / React
- VLM：Gemini 3.0 Flash (Phase 1a) → Qwen VL (Phase 1b)
- 数据库：PostgreSQL
- 任务调度：Celery

## 术语表

| 术语 | 含义 |
|------|------|
| 送审版 | 专家收到的待审材料（修改前），系统输入基准 |
| 审定版 | 专家评审后的修正版本（修改后），参考标准答案 |
| 三元关联 | 专家意见 ↔ 设计文件 ↔ 引用规范 的关联结构 |
| 必备件 | 所有项目必须包含的 7 类文件 |
| 可选件 | 项目间有差异的 4 类文件 |
